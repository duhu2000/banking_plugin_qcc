# Product Strategy 产品战略插件

> 基于 [Panaversity AgentFactory](https://github.com/panaversity/agentfactory-business-plugins) 的产品战略插件，集成企查查MCP服务，为企业提供产品规划、竞品分析和战略决策支持解决方案。

---

## 🎯 插件简介

Product Strategy 是一款面向产品管理和战略规划团队的AI智能体插件，覆盖产品全生命周期管理，从市场分析、产品规划、竞品对标到上市策略，帮助企业制定数据驱动的产品战略。

**核心理念**：用数据驱动产品决策，从市场中发现机会，在竞争中找到定位，实现产品的持续成功。

---

## 📋 包含功能模块

### 1. 市场分析
- **市场规模估算**：估算目标市场规模和增长潜力
- **市场细分**：识别细分市场和目标客户群
- **趋势分析**：分析市场趋势和变化
- **需求洞察**：挖掘用户痛点和需求
- **机会识别**：识别市场空白和机会点

### 2. 产品规划
- **产品路线图**：制定产品发展路线图
- **功能优先级**：基于价值评估功能优先级
- **版本规划**：规划版本发布计划
- **资源估算**：估算开发资源需求
- **里程碑管理**：跟踪产品开发里程碑

### 3. 竞品对标
- **竞品识别**：识别主要竞争对手
- **功能对标**：对比竞品功能特性
- **定价分析**：分析竞品定价策略
- **SWOT分析**：分析竞品优劣势
- **差异化定位**：找到产品差异化优势

### 4. 定价策略
- **成本分析**：分析产品成本结构
- **价值定价**：基于客户价值定价
- **竞争定价**：参考竞品定价
- **动态定价**：根据市场调整价格
- **促销策略**：制定促销和折扣策略

### 5. 上市策略
- **GTM计划**：制定产品上市计划
- **渠道策略**：选择分销渠道
- **营销方案**：规划营销推广活动
- **销售赋能**：准备销售工具包
- **上市检查**：产品上市前检查清单

### 6. 产品生命周期管理
- **导入期策略**：新产品上市策略
- **成长期策略**：快速增长阶段策略
- **成熟期策略**：市场饱和期策略
- **衰退期策略**：产品退市策略
- **产品退市**：管理产品生命周期结束

## 🚀 典型应用场景

### 场景1：新产品市场机会分析

```
📝 用户需求：
"评估供应链金融产品的市场机会"

⚡ 自动执行：
├── 分析目标市场：
│   ├── 中国供应链金融市场规模：15万亿元
│   ├── 年增长率：18%
│   ├── 目标客户：中小企业
│   └── 渗透率：仅20%
├── 识别市场痛点：
│   ├── 融资难：70%中小企业融资困难
│   ├── 成本高：传统融资成本10-15%
│   ├── 效率低：平均审批时间30天
│   └── 信息不对称：银行缺乏企业信用数据
├── 分析竞争格局：
│   ├── 主要竞品：10家银行、5家金融科技公司
│   ├── 产品特点：同质化严重
│   └── 定价：年化利率8-12%
├── 识别差异化机会
├── 估算市场潜力
└── 生成机会报告

📄 输出结果：
    ✅ 市场规模：15万亿元
    ✅ 目标市场：年融资需求3万亿元
    ✅ 可触达市场（TAM）：3000亿元
    ✅ 服务市场（SAM）：600亿元
    ✅ 可获得市场（SOM）：60亿元（3年）

    ✅ 市场痛点（优先级）：
        1. 融资难（影响度：高）
        2. 成本高（影响度：高）
        3. 效率低（影响度：中）

    ✅ 竞争分析：
        - 竞争优势：依托核心企业信用
        - 劣势：品牌知名度待建立
        - 机会：差异化定价策略
        - 威胁：金融科技公司技术领先

    ✅ 差异化策略：
        - 基于核心企业信用的低成本融资
        - 利用企查查数据快速授信
        - 目标利率：年化6-8%（低于竞品）
```

### 场景2：产品路线图规划

```
📝 用户需求：
"制定企业网银产品未来12个月路线图"

⚡ 自动执行：
├── 收集需求：
│   ├── 客户反馈：500条
│   ├── 销售需求：30项
│   ├── 竞品功能对标：50项
│   └── 技术可行性评估：20项
├── 功能分类：
│   ├── 基础功能：转账、查询
│   ├── 增值功能：批量支付、代发工资
│   ├── 创新功能：AI客服、智能风控
│   └── 生态功能：API开放、第三方集成
├── 优先级评分：
│   ├── 价值评分（40%）：客户价值、商业价值
│   ├── 紧迫度（30%）：市场竞争、客户需求
│   ├── 可行性（20%）：技术难度、资源需求
│   └── 风险（10%）：技术风险、合规风险
├── 排期规划
└── 生成路线图

📄 输出结果：
    ✅ 产品愿景：企业数字化金融管家
    ✅ 路线图（12个月）：

    Q1（基础夯实）：
    ├── 转账查询优化 ✅ 高优先级
    ├── 移动APP改版 ✅ 高优先级
    └── 安全认证升级 ✅ 高优先级

    Q2（效率提升）：
    ├── 批量支付功能 ✅ 高优先级
    ├── 代发工资服务 ✅ 中优先级
    └── 电子发票集成 ✅ 中优先级

    Q3（智能创新）：
    ├── AI智能客服 ✅ 高优先级
    ├── 智能风控预警 ✅ 高优先级
    └── 现金流预测 ✅ 中优先级

    Q4（生态开放）：
    ├── API开放平台 ✅ 中优先级
    ├── 第三方应用市场 ✅ 低优先级
    └── 行业解决方案 ✅ 低优先级

    ✅ 资源需求：开发团队50人
    ✅ 预算：2000万元
```

### 场景3：竞品定价分析

```
📝 用户需求：
"分析市场上企业贷款产品的定价策略"

⚡ 自动执行：
├── 识别主要竞品（10家银行）
├── 收集定价信息：
│   ├── 基准利率：LPR
│   ├── 浮动范围：+50bp 至 +300bp
│   └── 优惠条件：存款、结算量等
├── 分析定价模式：
│   ├── 成本加成法：3家银行
│   ├── 市场定价法：5家银行
│   └── 客户细分定价：2家银行
├── 计算实际利率分布
├── 分析定价与市场份额关系
└── 生成定价建议

📄 输出结果：
    ✅ 市场定价区间：年化 4.35% - 6.50%
    ✅ 平均定价：年化 5.20%
    ✅ 定价策略分析：
        ├── 国有大行：低定价（4.35-4.85%）+ 综合服务
        ├── 股份制银行：中定价（4.85-5.50%）+ 灵活产品
        └── 城商行：高定价（5.50-6.50%）+ 本地化服务

    ✅ 定价影响因素（按重要性）：
        1. 客户信用等级（权重30%）
        2. 存款贡献度（权重25%）
        3. 结算量（权重20%）
        4. 担保方式（权重15%）
        5. 期限（权重10%）

    ✅ 定价建议：
        - 基准定价：LPR + 100bp（5.20%）
        - 优质客户：LPR + 50bp（4.70%）
        - 普通客户：LPR + 100bp（5.20%）
        - 高风险客户：LPR + 200bp（6.20%）
```

## 🎬 与企查查MCP的潜在集成

### 产品战略规划

| Product Strategy 功能 | 企查查MCP增强 | 应用场景 |
|----------------------|--------------|----------|
| 竞品分析 | `qcc-company`: 查询竞争对手企业信息 | 分析竞品公司的注册资本、股东背景、经营状态 |
| 市场机会识别 | `qcc-operation`: 行业招投标数据分析 | 发现行业融资需求热点 |
| 客户细分 | `qcc-company`: 企业行业分布和规模 | 精准识别目标客户群体 |
| 产品定价 | `qcc-risk`: 企业信用等级数据 | 基于信用风险差异化定价 |
| 合作伙伴评估 | `qcc-company`: 评估生态合作伙伴 | 选择可靠的第三方服务商 |
