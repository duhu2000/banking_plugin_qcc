## Grading Schema

Each `grading.json` follows this structure:

```json
{
  "expectations": [
    {"text": "expectation text", "passed": true, "evidence": "supporting quote or reasoning"}
  ],
  "summary": {"passed": N, "failed": N, "total": N, "pass_rate": 0.XX}
}
```

Every expectation starts with a business-impact clause (e.g., "To prevent a budget cut that would destroy $378K in pipeline...") connecting the technical assertion to a revenue outcome.
